# 교차검증 모드 (Codex × 메인 Claude Cross-Check)

검증 대상: $ARGUMENTS

## 역할
하나의 질문/이슈/설계를 **두 독립 관점**으로 교차검증한다:
1. **Codex(gpt-5.6-sol)** — 외부 독립 엔진. **Bash로 `codex exec` 직접 호출** (pal/clink 불사용).
2. **메인 Claude(나, this 세션)** — 직접 코드/자료를 읽고 독립적으로 조사.

서로의 결론을 반박/동의/보완시켜 **합의된 결론과 남은 불일치**를 가려낸다.

## 언제 쓰나
- 단독 진단으로 확신이 안 서는 난제
- "왜/언제 이렇게 됐나"처럼 단정 위험이 큰 인과 추론
- 설계 선택지가 갈리거나, 보안 취약점 판단, 영향분석/업그레이드 리스크 평가
- 1차 진단 후 의견이 갈릴 때의 에스컬레이션 경로

가벼운 이슈엔 과하다. 교차검증은 **의도적으로** 발동하는 정밀 모드다.

## 비용·시간·권한
- codex exec 1콜 ≈ **15~24초** (2026-06-26 실측 기준; 직접 호출은 MCP 경유가 빠져 동등 이하). 구독 인증(`codex login` 상태) 재사용 — 별도 과금 없음.
- **콜 총 2회 고정** (R1 + R2). 메인 나의 직접 조사는 콜이 아니므로 추가 시간·한도 0.
- **권한: 전역 allowlist 등록됨(2026-07-03, ~/.claude/settings.json)** — `Bash(codex exec -s read-only:*)`, `Bash(codex exec resume:*)`. 어느 레포에서든 승인 프롬프트 없이 돈다.
- ⚠️ 접두사 매칭이므로 **`-s read-only`는 반드시 `codex exec` 바로 뒤에** 붙인다. 순서를 바꾸면 자동 허용이 깨져 승인 프롬프트가 뜬다 — 권한 룰이 곧 샌드박스 플래그 강제 장치다.

---

## read-only 원칙 ⚠️
- **1차 방어: `-s read-only`** — codex 자체 샌드박스가 OS 레벨에서 쓰기를 차단한다 (2026-07-03 Windows 실측 확인).
- **2차 방어: 프롬프트 첫 줄에 READ-ONLY 지시 유지** — codex가 쓰기를 시도조차 안 하게. (시도하면 에러가 조사 흐름을 흐린다.)

> "READ-ONLY 조사다. 파일을 **수정/생성/삭제하거나 명령으로 부수효과를 내지 마라.** 읽기·분석만 하고 결론을 보고하라."

코드/파일 수정은 **종합 단계에서 사용자 승인 후 메인 Claude(나)만** 수행한다.

## 1단계: 질문 구조화

검증 대상을 codex와 내가 **동일 기준**으로 답하도록 **공통 조사 프롬프트**를 만든다:

```
[READ-ONLY 조사 지시]

대상: {$ARGUMENTS을 명확히 재서술}
레포/경로: {관련 레포·파일 경로}

다음을 코드/설정/자료 근거와 함께 답하라:
1. 왜(원인) — 파일명:라인 등 출처 포함, 근본 원인
2. AS-IS — 현재 동작/상태
3. TO-BE — 올바른 동작/해결 방향
4. 영향 범위 — 무엇이 깨지나, 누가/무엇이 영향받나
5. 신뢰도 — 각 주장이 "근거로 확인됨" vs "추측"인지 반드시 구분

확인사실과 추정을 분리하고, 추정엔 "[추측]"을 붙여라.
```

- 대상이 **현재 cwd 밖의 레포**면 `-C <절대경로>`를 붙여 codex가 같은 코드를 보게 한다.
- **cwd가 git 레포가 아니면 `--skip-git-repo-check`를 붙인다** (2026-07-14 실측). 안 붙이면 codex가 `Not inside a trusted directory and --skip-git-repo-check was not specified.`를 내고 조사를 **실행조차 안 한다**. `-s read-only` **뒤에** 붙여야 allowlist 접두사 매칭이 안 깨진다. 레포 밖 경로를 `-C`로 가리킬 때도 그 경로가 레포가 아니면 동일하게 필요하다.

## 2단계: 라운드 1 — 진짜 병렬 독립 조사 (codex 1콜 + 나)

codex를 **백그라운드로** 띄우고, 도는 동안 내가 독립 조사한다:

```bash
# cwd가 git 레포가 아니면 -m 뒤에 --skip-git-repo-check 추가
codex exec -s read-only -m gpt-5.6-sol -o <세션 스크래치패드>/crosscheck_r1.txt - <<'EOF'
{1단계 공통 프롬프트 — READ-ONLY 첫 줄 포함}
EOF
```

- **`-m gpt-5.6-sol`은 반드시 `-s read-only` 뒤에** 온다. allowlist가 `codex exec -s read-only:*` 접두사 매칭이라, `-m`을 앞에 끼우면 자동 허용이 깨진다 (2026-07-10 실측).
- **cwd가 git 레포가 아니면 `--skip-git-repo-check`를 `-s read-only` 뒤에 붙인다** (2026-07-14 실측). 빠뜨리면 codex가 `Not inside a trusted directory and --skip-git-repo-check was not specified.`만 내고 조사를 실행하지 않는다.
- **run_in_background로 실행**한다. 여러 줄 프롬프트는 위처럼 **stdin(heredoc, `-` 인자)**으로 넘긴다 — 따옴표 이스케이프 사고 방지 (2026-07-03 실측 확인).
- codex가 도는 동안 **내가 직접** Read/Grep/Glob으로 같은 질문을 독립 조사한다.
- **블라인드 원칙: 내 결론을 확정하기 전에는 r1.txt를 열지 않는다.** (clink 시절엔 순서로만 보장했지만 이제 진짜 병렬이다.)
- **codex 생사 판단**: exit code ≠ 0이거나 r1.txt가 없거나 빈 파일이면 즉시 명시하고 **나 단독으로 진행**한다. 절대 조용히 누락하지 않는다.

## 3단계: 라운드 2 — 상호 반박/교차검증 (codex 1콜 + 나)

**codex에게 내 결론을 보여주고** 반박/동의/보완을 받는다. R1 세션을 이어서:

```bash
# R1에서 --skip-git-repo-check를 썼으면(비레포 cwd) R2에도 동일하게 붙인다
codex exec resume --last -c 'sandbox_mode="read-only"' -o <세션 스크래치패드>/crosscheck_r2.txt - <<'EOF'
[READ-ONLY] 다른 조사자(메인 Claude)의 결론이다:

[메인 Claude]: {내 요지 + 핵심 근거(파일:라인 등)}

너의 기존 결론과 비교해 답하라:
1. 동의하는 부분 / 반박하는 부분 (각각 근거)
2. 상대가 놓친 것 / 네가 틀렸던 것
3. 최종 입장 (바뀌었다면 왜)
4. 여전히 확신 못 하는 지점
EOF
```

- `resume`엔 `-s` 플래그가 없다 → **`-c 'sandbox_mode="read-only"'` 오버라이드 필수** (2026-07-03 실측: `-s` 넣으면 usage 에러).
- **모델은 R1 세션에서 자동 상속된다 → R2엔 `-m`을 붙이지 마라** (2026-07-10 실측: 그냥 `resume --last`가 `gpt-5.6-sol` 유지). 접미사 없는 `-m gpt-5.6`을 R2에 끼우면 오히려 400 차단.
- `--last`는 같은 cwd의 최신 세션을 잡는다 → **R1과 R2 사이에 다른 codex 실행을 하지 마라.**
- **메인 나**: codex의 R1 결론을 받아 **내가 직접 반박/검증**한다. codex가 든 근거(파일:라인 등)를 실제로 열어 확인한다. codex가 옳으면 내 입장을 바꾸고, 틀렸으면 근거로 반박한다.

**원칙**: "다수결"이 아니라 **근거의 질**로 판단한다. codex와 나 1:1이므로, **내가 양쪽 근거를 직접 검증**해 최종 판정한다(내가 심판이자 당사자이므로, 내 편을 들지 않도록 codex 근거를 더 엄격히 검증한다).

> codex가 R1에서 죽었다면 R2는 생략하고 4단계로 간다(반박 상대 없음). 이땐 내 결론을 스스로 적대적으로 재검증하는 것으로 대신한다.

## 4단계: 종합 (메인 Claude)

교차검증 결과를 종합한다. **codex 의견을 그대로 베끼지 않고**, 근거를 직접 검증(재확인)한 뒤 결론낸다.

### 출력 형식 (화면 출력)

```
## 교차검증 요약: {대상}

### 참여 관점
- Codex(gpt-5.6-sol) / 메인 Claude (codex 실패 시 명시. gemini·claude 외부호출은 상시 제외)

### 합의된 결론 ✅
| 항목 | 결론 | 신뢰도 | 동의 |
|------|------|--------|------|
| 근본 원인 | ... | 근거 확인 | codex,나 |
| ... | ... | ... | ... |

### 불일치 / 미해결 ⚠️
| 쟁점 | Codex | 메인 Claude | 내 최종 판단 |
|------|-------|-------------|--------------|
| ... | ... | ... | ...+근거 |

### 최종 결론
{내가 근거를 직접 검증해 내린 결론. 합의가 약한 부분은 명시}

### 해결 방안
{수정 위치(파일:라인 등), 우선순위. "~하면 이렇게 동작한다" 형식}

### 남은 검증 과제
{실측/확인이 더 필요한 것 — 단정 금지}
```

## 종료 기준
- **합의된 것**과 **갈린 것**이 표로 분리됐다
- 갈린 쟁점마다 내가 근거의 질로 판단을 내렸다 (또는 "실측 필요"로 남김)
- 모든 인과 주장이 "확인됨/추측"으로 분류됐다
- 코드/파일 수정은 제안만 — 적용은 사용자 승인 후 메인 Claude가 수행

## 주의사항
- codex가 "왜 이렇게 됐나"를 단정해도, 내가 검증 못 한 건 "[추측]"으로 내려쓴다. "정말?" 소리 들으면 즉시 실측.
- codex 응답(r1/r2.txt)이 길면 결론·근거만 추출한다(원문 전체 인용 금지).
- **gemini·claude 외부호출은 상시 제외.** gemini는 무료 차단, claude는 메인 나와 중복. 호출 시도조차 하지 마라.
- codex 콜은 2회를 넘기지 마라. 더 줄이려면 R2를 생략하고 내가 codex R1을 직접 검증해도 된다.
- 토큰·시간이 드는 모드다. 가벼운 건 이 모드로 발동하지 말고 단독 진단으로 처리한다.

### 3번째 관점? — 저사양 노트북에선 비추천
완전 독립된 3번째 엔진으로 로컬 모델을 생각할 수 있으나, **전용 GPU 없는 노트북(통합그래픽 + RAM 16GB급)에선 부적합**(2026-06-26 실측):
- 돌릴 수 있는 건 3~8B급뿐(30B MoE는 RAM 초과라 불가). CPU 추론이라 **초당 몇 토큰 → crosscheck 콜이 수십초~분**, 속도 이점 없음.
- 3~8B는 gpt-5.6-sol/opus 대비 추론력 격차 커 **난제 교차검증의 "강한 눈" 역할 불가**.
- **결론: 2모델 체제(codex × 메인나)가 이 하드웨어의 최적해.** 로컬 추가는 GPU 워크스테이션 생기면 재검토.

## 설계 근거 / 연혁 (2026-06-26 확정, 2026-07-03 pal 제거, 2026-07-10 gpt-5.6-sol 승격)
- **모델 gpt-5.6-sol 채택 (2026-07-10 실측)**: 5.6부터 티어제(sol 프런티어 / terra 저비용 / luna 고속)로 바뀌었고, `sol`은 코드네임이 아니라 정식 모델명의 일부다. **접미사 없는 `gpt-5.6`은 ChatGPT 구독 인증에서 400 차단**(`not supported when using Codex with a ChatGPT account`)이라 "sol 뗀 5.6"을 기다리는 건 무의미 — `gpt-5.6-sol`이 구독으로 쓸 수 있는 프런티어 티어다. codex-cli **0.144+ 필요**(0.142는 `requires a newer version` 400). R1에 `-m gpt-5.6-sol`만 붙이면 R2 `resume --last`는 세션에서 모델을 **자동 상속**한다(실측: R2에 모델 플래그 불필요).
- **pal:clink 제거 (2026-07-03 실측)**: clink는 codex CLI를 셸로 띄우는 래퍼일 뿐이다. 직접 호출로 바꾸면 ① 의존성 1개 제거 ② `-s read-only`가 **OS 레벨 샌드박스**로 강제됨(Windows 실측: 쓰기 시도 → UnauthorizedAccessException 차단, 파일 미생성. clink는 `--dangerously-bypass`로 띄우고 프롬프트로만 부탁하는 구조였다) ③ `exec resume`이 continuation_id를 완전 대체.
- **gemini**: 2026-06-18부로 무료 OAuth 차단(`UNSUPPORTED_CLIENT`). 헤드리스 자동화 불가 → **제외**.
- **claude 외부호출 폐기 유지**: clink claude든 `claude -p`든 결국 **메인 나와 같은 모델**을 시간·한도 써가며 한 번 더 부르는 것. 독립 관점 가치가 없다. → 그 자리는 메인 나의 직접 조사.
- 결과: codex(진짜 외부 엔진) + 나(직접 검증) = **독립성 유지 + 의존성 최소 + 진짜 샌드박스**.
