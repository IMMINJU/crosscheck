# 교차검증 모드 (Codex × 각도 서브에이전트 × 메인 Claude)

검증 대상: $ARGUMENTS

## 0. 구조·원칙 요약

하나의 질문/이슈/설계를 **독립성 2축(모델 축 × 각도 축)**으로 교차검증해, 반박/동의/보완을 거쳐 **합의된 결론과 남은 불일치**를 가려낸다.

**흔한 "다른 모델에게 리뷰 시키기"(한방향)와 다른 점 4가지** — 이 프로토콜의 정체성:
1. **블라인드 병렬 독립조사** — 서로의 결론을 보기 전에 각자 결론부터 확정한다 (앵커링 차단)
2. **상호반박 라운드(R2)** — 한방향 리뷰가 아니라 양방향 반박과 입장 갱신
3. **근거 직접 재검증** — 심판이 인용된 파일:라인을 실제로 열어 확인한 것만 채택한다 (다수결 배제)
4. **확인/[추측] 라벨 강제** — 모든 인과 주장을 검증 상태로 분류한다

왜 이 구조인가: 타 벤더 모델 = self-preference·오류 상관 회피 / 각도 분해 = 중복이 아닌 "상이한 측면의 병렬 탐색" / 블라인드·라운드 고정 = 앵커링·사이코팬시 수렴 방지. (상세는 부록 연혁)

| 참여자 | 하는 일 | 독립성 |
|--------|---------|--------|
| **Codex(gpt-5.6-sol)** | **전체 질문 1세션** — R1 독립조사 + R2 반박. Bash로 `codex exec` 직접 호출 (pal/clink 불사용) | 모델 축: 다른 모델 = 오류 탈상관 |
| **각도 서브에이전트 2~5개** (Claude, Explore 타입) | 질문을 **직교 각도로 분해**해 병렬 조사. R1만 참여 | 각도 축 |
| **메인 Claude(나)** | 직접 조사하는 당사자 + 모든 진영의 근거를 재검증하는 심판 | — |

철칙 5개:
1. **같은 질문을 같은 모델에 그대로 중복시키지 않는다** — 오류 상관이 높아 무가치. 중복이 아니라 분해다.
2. **블라인드** — 내 결론을 확정하기 전에는 r1.txt도 각도 보고서도 열지 않는다.
3. **다수결이 아니라 근거의 질** — 인용된 근거(파일:라인)를 내가 실제로 열어 확인한 것만 채택한다. 내가 심판이자 당사자이므로, 내 편을 들지 않도록 codex 근거를 더 엄격히 검증한다.
4. **READ-ONLY 조사** — 조사 단계에선 아무도 파일을 건드리지 않는다. 수정은 종합 단계에서 사용자 승인 후 메인 나만.
5. **합의는 증거가 아니다** — 프런티어 모델들은 같은 오답에 수렴한다(에러 상관: 둘 다 틀릴 때 같은 오답 ~60% 일치, 프런티어일수록 상관이 더 높음 — Kim 2025 / 9-judge 패널의 유효 독립표 2.18표 — Kohli 2026). codex와 내가 일치해도 근거 재검증을 면제하지 않는다. 합의표의 ✅는 "합의라서 통과"가 아니라 "근거 재검증을 통과한 것만 올리는 자리"다.

### 발동/생략 기준
- **쓴다**: 단독 진단으로 확신이 안 서는 난제 / "왜·언제 이렇게 됐나"류 단정 위험 큰 인과 추론 / 설계 선택 분기·보안 취약점 판단·영향분석·업그레이드 리스크 / 1차 진단 후 의견이 갈릴 때의 에스컬레이션.
- **안 쓴다**: 가벼운 이슈. **의도적으로** 발동하는 정밀 모드다 — 풀 구성은 토큰이 단독 진단의 십수 배까지 든다.
- **각도 생략**: 단일 쟁점의 좁은 질문이면 서브에이전트 없이 기존 2관점(codex+나)으로 돈다. 직교 각도가 억지스러우면 만들지 마라.

### 비용·한도
- codex exec 1콜 ≈ **실전 조사 60~217초** (2026-07-07 실측 교정; 초기 "15~24초"는 미니 프롬프트 기준이라 폐기). 구독 인증(`codex login` 상태) 재사용 — 별도 과금 없음.
- **codex 콜 총 2회 상한** (R1+R2). R2는 조건부 — 발동 기준은 §2-3 (결론+근거까지 동일하면 생략). 각도 서브에이전트는 Claude 토큰만 쓰고(codex 한도 0) codex R1과 병렬이라 벽시계 시간도 겹친다. 메인 나의 직접 조사는 콜이 아니므로 추가 0.
- **다중 세션 동시 사용 가능 (2026-07-20 실측)**: resume이 UUID 타겟이고 산출물(r1/r2.txt)이 세션 스크래치패드에 격리되므로, 여러 Claude Code 세션이 동시에 이 모드를 돌려도 충돌하지 않는다. codex 프로세스 동시 실행도 무해(모델캐시 갱신 경합 ERROR 로그는 exit 0이면 무시). 유일한 예외: `--last` 폴백만 다중 세션에서 위험(§2-3 폴백 체인 준수). 구독 한도는 세션들이 공유한다.

## 1. 커맨드 철칙 (전 단계 공통 — 어기면 승인 프롬프트가 뜨거나 400)

전역 allowlist(~/.claude/settings.json, 2026-07-03): `Bash(codex exec -s read-only:*)` + `Bash(codex exec resume:*)`. **접두사 매칭이라 플래그 순서가 곧 read-only 강제 장치다.**

| 규칙 | 근거·실측 |
|------|-----------|
| `-s read-only`는 `codex exec` **바로 뒤에** | 순서 바꾸면 allowlist 미매칭 → 승인 프롬프트 (2026-07-03) |
| `-m gpt-5.6-sol`은 `-s read-only` **뒤에** (R1에만) | `-m`을 앞에 끼우면 매칭 깨짐 (2026-07-10 실측) |
| R2(resume)엔 `-m` **금지** | 모델은 R1 세션에서 자동 상속(UUID resume도 동일, 2026-07-20 실측). 접미사 없는 `gpt-5.6`은 400 차단 (2026-07-10 실측) |
| resume엔 `-s` 플래그가 없다 → `-c 'sandbox_mode="read-only"'` 오버라이드 **필수** | `-s` 넣으면 usage 에러 (2026-07-03 실측) |
| cwd가 git 레포가 아니면 `--skip-git-repo-check`를 `-s read-only` **뒤에** (R1·R2 동일) | 없으면 `Not inside a trusted directory` 내고 실행 자체를 안 함 (2026-07-14 실측). `-C <절대경로>`로 가리킨 경로가 비레포여도 동일 |
| 여러 줄 프롬프트는 **heredoc stdin(`-` 인자)** | 따옴표 이스케이프 사고 방지 (2026-07-03 실측) |
| read-only **이중방어**: `-s` 샌드박스(OS 레벨 쓰기 차단, Windows 실측) + 프롬프트 첫 줄 READ-ONLY 지시 | 쓰기 시도 에러가 조사 흐름을 흐리는 것도 방지 |

READ-ONLY 지시문 (codex·서브에이전트 공통, 프롬프트 첫 줄):
> "READ-ONLY 조사다. 파일을 **수정/생성/삭제하거나 명령으로 부수효과를 내지 마라.** 읽기·분석만 하고 결론을 보고하라."

## 2. 절차

### 2-1. 질문 구조화 + 각도 분해

codex와 내가 **동일 기준**으로 답하도록 **공통 조사 프롬프트**를 만든다:

```
[READ-ONLY 조사 지시]

대상: {$ARGUMENTS을 명확히 재서술}
레포/경로: {관련 레포·파일 경로}

다음을 코드/설정/자료 근거와 함께 답하라:
1. 왜(원인) — 파일명:라인 등 출처 포함, 근본 원인
2. AS-IS — 현재 동작/상태
3. TO-BE — 올바른 동작/해결 방향
4. 영향 범위 — 무엇이 깨지나, 누가/무엇이 영향받나
5. 신뢰도 — 각 주장이 "근거로 확인됨" vs "추측"인지 반드시 구분

확인사실과 추정을 분리하고, 추정엔 "[추측]"을 붙여라.
```

- 대상이 **현재 cwd 밖의 레포**면 `-C <절대경로>`를 붙여 codex가 같은 코드를 보게 한다.

**각도 분해 (Claude 서브에이전트용)**: 같은 질문을 **직교하는 각도 2~5개**로 분해한다. 겹치는 각도는 합치고, 억지로 늘리지 않는다 — **패러프레이즈 N개는 다양성이 아니다**. 기본 축(질문에 맞게 취사·변형):
1. **코드 정적** — 현재 구현이 실제로 어떻게 동작하나 (파일:라인)
2. **이력** — git log/blame: 언제·왜 이렇게 됐나
3. **문서·스펙·외부** — 의도된 동작은 무엇인가 (공식문서/이슈트래커/웹)
4. **적대·반례** — 유력 가설을 깨는 시나리오·엣지케이스만 탐색
5. **환경·재현** — 설정/버전/플랫폼 차이, 재현 조건

각 서브에이전트 프롬프트에 반드시: READ-ONLY 첫 줄 + **각도 목표** + **경계("~는 다루지 마라" — 옆 각도와 중복 금지)** + 출력 형식(결론 / 근거 파일:라인 / "확인 vs [추측]" 구분). 경계 없는 막연한 지시는 중복 조사를 낳는다.

### 2-2. 라운드 1 — 전원 병렬 발사 + 내 독립 조사

```bash
# 비레포 cwd면 -m 뒤에 --skip-git-repo-check 추가 (§1)
codex exec -s read-only -m gpt-5.6-sol -o <세션 스크래치패드>/crosscheck_r1.txt - <<'EOF'
{2-1 공통 프롬프트 — READ-ONLY 첫 줄 포함}
EOF
```

- **run_in_background로 실행**하고, 출력 배너의 **`session id: <uuid>` 라인을 기록**한다 — R2 resume 타겟.
- **각도 서브에이전트 동시 발사**: Agent 툴을 **한 메시지에 N개** 담아 병렬 실행한다(기본 백그라운드, Explore 타입 — Edit/Write 자체가 없음). 프롬프트는 2-1 각도 규격대로.
- 전부 도는 동안 **내가 직접** Read/Grep/Glob으로 같은 질문을 독립 조사하고 결론을 확정한다. **확정 전에는 r1.txt도 각도 보고서도 열지 않는다** (블라인드).
- **생사 판단**: codex가 죽으면(exit code ≠ 0, r1.txt 없음/빈 파일) 즉시 명시하고 나+각도 서브에이전트로 진행한다. 죽은 각도 에이전트도 명시하고 그 각도는 종합 단계에서 내가 직접 메꾼다. 절대 조용히 누락하지 않는다.

### 2-3. 라운드 2 — 상호 반박/교차검증

**R2 발동 기준** — 하나라도 해당하면 실행:
- 핵심 원인/결론에 진영 간 불일치
- 결론은 같은데 **핵심 근거가 다름**
- 한쪽만 [확인], 다른 쪽은 [추측]
- 적대 반례가 결론을 실제로 위협
- **고위험 사안(보안·데이터 손실·마이그레이션)은 합의 여부와 무관하게 강제**

전부 아니면(결론+핵심 원시 근거까지 동일) **R2 생략** — 라운드 추가는 정확도를 높이지 않고 이미 맞은 결론을 흔들 수 있다(Choi NeurIPS 2025: 이득의 본체는 독립 응답, 토론이 그걸 갉아먹음). 단, 생략해도 "같은 결론이 같은 모델 편향의 산물일 수 있음"을 종합에서 명시한다(철칙 5).

**R2 프롬프트는 익명 증거 패킷** — 진영명("메인 Claude 진영")·합의 인원수("N개 에이전트가 동의")는 **수적 압력으로 작동하므로 절대 전달 금지**(Conformity in LLMs 2024: 틀린 다수 의견에 동조). 쟁점별로 주장과 원시 근거만 익명으로 담고, codex 자신의 R1 입장을 먼저 고정 인용한다. 패킷을 만들 때 **상충 근거를 빠뜨리지 않는다** — 메인의 요약 선별이 새로운 편향이 되지 않게.

R1 세션을 **세션 ID로** 이어서:

```bash
# R1에서 --skip-git-repo-check를 썼으면(비레포 cwd) R2에도 동일하게 붙인다
codex exec resume <R1 배너의 session id> -c 'sandbox_mode="read-only"' -o <세션 스크래치패드>/crosscheck_r2.txt - <<'EOF'
[READ-ONLY] 교차검증 라운드 2다. 아래는 쟁점별 대안 주장과 원시 근거다. 누가/몇 명이 주장했는지는 의도적으로 제공하지 않는다 — 수적 합의나 출처 권위를 신호로 취급하지 말고 근거만 평가하라.

너의 R1 입장 (고정): {codex r1.txt의 쟁점별 결론 요지}

쟁점별 대안 주장 (익명):
- 쟁점 1: {주장} / 근거: {파일:라인, 로그 등 원시 근거}
- 쟁점 2: ...

각 쟁점에 답하라:
1. R1 입장: 유지 / 수정 / 기권
2. 수정한다면 — 어떤 **새 직접 증거**가 네 기존 근거를 반증하는가. 증거 없이 입장을 바꾸지 마라.
3. 대안 주장의 근거에서 틀렸거나 불충분한 점
4. 여전히 확신 못 하는 지점
EOF
```

- **세션 ID resume이 기본** (2026-07-20 실측: 병렬 세션 중 비최신 세션도 UUID로 정확 타겟, allowlist 그대로 커버). ID를 잃었을 때 폴백 체인: ① **rollout 파일명에서 UUID 복구** — `~/.codex/sessions/YYYY/MM/DD/rollout-<시각>-<uuid>.jsonl`에 프롬프트 원문이 남으므로 내 R1 프롬프트 문구를 Grep해 세션 식별(실측) ② `--last`는 최후 수단 — 같은 cwd의 최신 세션을 잡으므로 **다른 codex 실행(다른 Claude 세션 포함)이 끼어들었을 수 있으면 금지.**
- **메인 나**: codex R1 결론과 각도 보고서를 받아 **내가 직접 반박/검증**한다. 서브에이전트 주장도 codex와 동급의 미검증 주장이다 — 인용된 근거(파일:라인 등)를 실제로 열어 확인한 것만 채택. codex가 옳으면 내 입장을 바꾸고, 틀렸으면 근거로 반박한다.
- codex가 R1에서 죽었다면 R2는 생략하고 종합으로 간다(반박 상대 없음). 이땐 내 결론을 스스로 적대적으로 재검증하는 것으로 대신한다.

### 2-4. 종합·출력 (메인 Claude)

**codex·서브에이전트 의견을 그대로 베끼지 않고**, 근거를 직접 재확인한 뒤 결론낸다.

**핵심 인과 주장(근본 원인·영향 범위·"수정하면 해결된다")에는 확인 라벨을 3단계로 세분화한다** — 파일:라인 재확인은 존재 검증이지 인과 검증이 아니다:
- `[확인:존재]` — 그 코드/설정/문서가 실재함 (파일:라인으로 충분)
- `[확인:경로]` — 문제 상황에서 그 경로가 실제로 도달·적용됨
- `[확인:인과]` — 로그·재현·변경 전후 차이·경쟁 가설 배제 중 하나 이상으로 인과가 입증됨

`[확인:존재]`만으로 인과 결론을 "확인"으로 승격 금지 — `[확인:경로]` + 독립 보조 근거가 없으면 `[추측]`으로 내려쓴다. 단순 사실 주장에는 기존 확인/[추측] 2단계 그대로 (전부 3단계로 쓰면 READ-ONLY 조사에선 모든 게 [추측]이 된다).

화면 출력 형식:

```
## 교차검증 요약: {대상}

### 참여 관점
- Codex(gpt-5.6-sol, 전체질문 1세션) / 각도 서브에이전트({각도명 나열}) / 메인 Claude (죽은 참여자는 명시)

### 합의된 결론 ✅
| 항목 | 결론 | 신뢰도 | 동의 |
|------|------|--------|------|
| 근본 원인 | ... | 근거 확인 | codex,나 |
| ... | ... | ... | ... |

### 불일치 / 미해결 ⚠️
| 쟁점 | Codex | 메인 Claude | 내 최종 판단 |
|------|-------|-------------|--------------|
| ... | ... | ... | ...+근거 |

### 입장 전환 감사 (R2 실행 시)
| 쟁점 | Codex R1 | 메인 R1 | R2 후 | 변경 유발 근거 | 분류 |
|------|----------|---------|-------|----------------|------|
| ... | ... | ... | ... | ... | ... |

분류: **새 증거에 따른 정당한 갱신** / **근거 없는 합의 수렴(⚠️경고)** / **초기 근거가 더 강했는데 약화됨(⚠️경고)** / **판정 불가**. R1 원답은 지우지 않고 반드시 이 표에 보존한다 — 최종 결론만 남기면 R2가 오류를 고쳤는지, 맞았던 소수 의견을 지웠는지 사후 감사가 불가능하다.

### 최종 결론
{내가 근거를 직접 검증해 내린 결론. 합의가 약한 부분은 명시}

### 해결 방안
{수정 위치(파일:라인 등), 우선순위. "~하면 이렇게 동작한다" 형식}

### 남은 검증 과제
{실측/확인이 더 필요한 것 — 단정 금지}
```

종료 기준:
- **합의된 것**과 **갈린 것**이 표로 분리됐다
- 갈린 쟁점마다 내가 근거의 질로 판단을 내렸다 (또는 "실측 필요"로 남김)
- 모든 인과 주장이 "확인됨/추측"으로 분류됐다
- 코드/파일 수정은 제안만 — 적용은 사용자 승인 후 메인 Claude가 수행

## 3. 금지·상한

- **gemini·claude 외부호출 상시 제외.** gemini는 무료 차단, claude는 메인 나와 중복. 호출 시도조차 하지 마라.
- codex 콜 2회 초과 금지. 각도는 최대 5개 — 각도 보고서도 근거 재검증 대상이라, 각도 수만큼 종합 단계의 검증 부담이 는다.
- codex 응답(r1/r2.txt)이 길면 결론·근거만 추출한다(원문 전체 인용 금지).
- codex가 인과를 단정해도 내가 검증 못 한 건 "[추측]"으로 내려쓴다. "정말?" 소리 들으면 즉시 실측.

## 부록: 설계 연혁 (실행에는 불필요 — 배경 기록)

(2026-06-26 확정, 2026-07-03 pal 제거, 2026-07-10 gpt-5.6-sol 승격, 2026-07-20 각도분산+UUID resume, 2026-07-23 MAD 연구 반영)

- **MAD(multi-agent debate) 부정 연구 반영 (2026-07-23)**: "모델 경쟁이 오답률을 높인다"는 주장을 Claude 웹조사 + codex 독립조사로 교차검증(이 스킬을 스킬 자신에게 적용). 결론: 실패하는 건 **합의 강제 + 반복 토론 + 근거 검증 없는 주장 대결** 구조(Wynn 2509.05396: 정답→오답 전환이 역방향보다 잦음, 혼합구성 MMLU −12%p / Choi NeurIPS 2025: MAD 이득의 본체는 토론이 아니라 초기 독립 응답, 토론 추가 시 이득 소멸 / Zhang 2502.08788: MAD가 CoT·self-consistency 못 이김, 유일한 일관 개선책=모델 이질성 / Khan ICML 2024: 검증 가능한 근거+강한 판정자일 때만 토론이 작동, judge 48%→76%). 기존 구조(블라인드 R1·다수결 배제·근거 재검증·2라운드 상한·모델 이질성)는 이미 안전지대. 이번 개정 5개: ① 철칙 5 "합의는 증거가 아니다"(에러 상관: Kim 2506.07962 같은 오답 60% 일치, Kohli 2605.29800 9-judge 유효표 2.18) ② R2 익명 증거 패킷(진영명·합의 수 제거 — Conformity 2410.12428) ③ 인과 라벨 3단계(존재/경로/인과 — 파일:라인은 존재 검증이지 인과 검증이 아님, codex 발견) ④ R1→R2 전환 감사표(원답 보존) ⑤ R2 조건부 발동. 부수 확인: Claude 조사와 codex 조사의 맹점이 정확히 갈렸음(Claude=에러 상관 축, codex=검증 사다리 축) — 이 스킬의 존재 이유 실증.

- **각도 분산 구조 + UUID resume (2026-07-20 실측)**: 라운드1을 "codex 전체질문 1세션(모델 축) + Claude 각도 서브에이전트 병렬(각도 축)"로 확장. 근거: 같은 모델에 같은 질문을 중복시키면 오류가 상관돼 무가치하고, 이득은 각도 분해에서 나온다(Anthropic 멀티에이전트 리서치 시스템 — 막연한 지시로 서브에이전트 2개가 중복 조사한 실패사례, "각 서브에이전트에 목표·출력형식·경계 필요", 90.2% 우위의 메커니즘 = "서로 다른 측면의 병렬 탐색"). R2는 `codex exec resume <uuid>`로 전환 — 병렬 세션 중 비최신 세션 UUID 타겟팅·모델 자동 상속·기존 allowlist 커버·rollout 파일명 폴백 전부 실측 확인. 부수 관찰: codex 2개 병렬 시 모델캐시 갱신 경합 ERROR 로그가 뜰 수 있으나 exit 0이면 무해. 파생 효과: `--last` 의존이 사라져 **여러 Claude Code 세션의 동시 crosscheck이 안전**해졌다(산출물은 원래부터 세션 스크래치패드 격리).
- **모델 gpt-5.6-sol 채택 (2026-07-10 실측)**: 5.6부터 티어제(sol 프런티어 / terra 저비용 / luna 고속)로 바뀌었고, `sol`은 코드네임이 아니라 정식 모델명의 일부다. **접미사 없는 `gpt-5.6`은 ChatGPT 구독 인증에서 400 차단**(`not supported when using Codex with a ChatGPT account`)이라 "sol 뗀 5.6"을 기다리는 건 무의미 — `gpt-5.6-sol`이 구독으로 쓸 수 있는 프런티어 티어다. codex-cli **0.144+ 필요**(0.142는 `requires a newer version` 400). R1에 `-m gpt-5.6-sol`만 붙이면 R2 resume은 세션에서 모델을 **자동 상속**한다.
- **pal:clink 제거 (2026-07-03 실측)**: clink는 codex CLI를 셸로 띄우는 래퍼일 뿐이다. 직접 호출로 바꾸면 ① 의존성 1개 제거 ② `-s read-only`가 **OS 레벨 샌드박스**로 강제됨(Windows 실측: 쓰기 시도 → UnauthorizedAccessException 차단, 파일 미생성. clink는 `--dangerously-bypass`로 띄우고 프롬프트로만 부탁하는 구조였다) ③ `exec resume`이 continuation_id를 완전 대체.
- **gemini**: 2026-06-18부로 무료 OAuth 차단(`UNSUPPORTED_CLIENT`). 헤드리스 자동화 불가 → **제외**.
- **claude 외부호출 폐기 유지**: clink claude든 `claude -p`든 결국 **메인 나와 같은 모델**을 시간·한도 써가며 한 번 더 부르는 것. 독립 관점 가치가 없다. → 그 자리는 메인 나의 직접 조사 + 각도 분해다.
- **3번째 엔진(로컬 모델)? — 저사양 노트북에선 비추천 (2026-06-26 실측)**: 전용 GPU 없는 노트북(통합그래픽 + RAM 16GB급)에선 3~8B급만 가능(30B MoE는 RAM 초과). CPU 추론이라 초당 몇 토큰 → 콜이 수십초~분, 속도 이점 없음. 3~8B는 gpt-5.6-sol/opus 대비 추론력 격차 커 난제 교차검증의 "강한 눈" 역할 불가. **2모델 체제(codex × 메인나)가 이 하드웨어의 최적해.** 로컬 추가는 GPU 워크스테이션 생기면 재검토.
- 결과: codex(진짜 외부 엔진) + 각도 서브에이전트(질문 분해) + 나(직접 검증) = **독립성 2축 + 의존성 최소 + 진짜 샌드박스**.
